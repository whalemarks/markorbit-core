# Core Domain Contract Skeletons

Core Domain Contract Skeletons start the explicit domain contract layer for Book 02 — MarkOrbit Core Specification. They create one lightweight contract boundary for each baseline Core Domain so the contract index can name the domain layer before later tasks expand the details.

These skeletons are intentionally not full domain specifications. They do not define business objects, object schemas, services, APIs, workflows, workflow runtime behavior, database schema, product UI, or AI agent authority.

Every baseline Core Domain in the canonical Core Domain Registry must have exactly one skeleton. Codex may not add, remove, or rename baseline domains without explicit approval.

Capability and Business Responsibility remain cross-cutting systems, not baseline domains. Execution Context and Execution Runtime belong to Book 03, not Book 02 domain contracts. Artifact, Render, Publish, Distillery, and Workplace are not baseline domain contracts in this task.

Future tasks may expand each domain skeleton into a full domain contract file while preserving the Core / Execution / Product boundaries established here.
