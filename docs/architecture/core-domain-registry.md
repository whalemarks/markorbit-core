# Core Domain Registry

The Core Domain Registry is the canonical typed list of MarkOrbit Core domains. It exists to provide a stable, explicit baseline for Core domain identifiers, display names, categories, descriptions, source book metadata, and activation status before any domain business logic is implemented.

The registry comes from Book 02 — MarkOrbit Core Specification and represents the baseline Core domain set for this repository. The baseline contains exactly 26 Core Domains grouped into foundation, professional, business execution, and collaboration network categories.

Codex may not add, rename, or remove Core domains without explicit approval. New product concepts, execution concepts, research concepts, or cross-cutting systems must not be promoted into this registry by inference.

Capability and Business Responsibility are cross-cutting systems, not baseline Core Domains. Execution Context and Execution Runtime belong to Book 03 and are not Core domains. Artifact, Render, Publish, and Distillery are not part of this baseline registry.
